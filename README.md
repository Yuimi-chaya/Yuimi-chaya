## 关于代码

我的主要开发方式是 **Vibe Coding**。

大致流程如下：

1. 我产生一个想法。
2. AI 表示完全理解。
3. AI 开始写代码。
4. 我在旁边随缘看着。
5. 程序居然运行了。
6. 我们都不知道为什么。

我负责描述感觉、点击运行，以及在事情失控时要求它“再仔细检查一下”。

> 专业程度：取决于 AI 今天的状态。

## 关于博客

文章没有写多少，主题倒是改过很多次。

经常花几个小时调整字体、间距、颜色，以及一些可能根本没人注意的细节。

写博客是次要的。

让博客看起来像一个真正属于自己的地方，才比较重要。

---

## 关于 LLM RP

偶尔尝试让一团概率分布表现得像一个真实存在的人。

研究谈不上，主要是在角色设定、提示词、记忆、情绪和上下文之间反复折腾，然后观察它什么时候突然不再像人。

有时效果很好。

有时它会用三段话重复刚才说过的内容。

---

## 关于 MMD

稍微知道怎么让角色动起来。

会一点动作、镜头和时间轴，然后在“差不多能看”和“这里好像不太对”之间不断调整。

距离专业制作还很远。

不过看到喜欢的角色真的按照自己的想法动起来，已经足够有意思了。
